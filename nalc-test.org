* CTF Plan Summary

** File format
=.ctf= files (not =.el= — calc display like =2:  3= isn't valid elisp, so a
dedicated extension avoids read errors).
One file = one ERT test.

** Parser
State machine that uses =read= for s-expressions and switches to raw-text
capture after =(ctf-declare-stack)= / =(ctf-declare-output)=, collecting
lines until the next =(=.
Stack lines (=N:  expr=) are parsed with =math-read-expr=.

** Key semantics
- ="H"= in point specs = home (=.= line)
- Stack entries pushed bottom-first via =calc-push=
- Output compared with =math-equal= / =math-normalize=
- =ctf-declare-start-cond= / =ctf-declare-end-cond= forms are evaled and
  asserted

** ERT integration
=ctf-register-file= parses a =.ctf= file and evals an =ert-deftest= form.
=ctf-load-directory= is called from =test-init.el= so CTF tests run
alongside existing ERT tests with no change to the test runner invocation.

** Single new file
=my/calc/tests/ctf.el= — parser + runner + ERT registration.
