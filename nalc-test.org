#+title: nalc-test

* File format
=.nalc-test= files (not =.el= — calc display like =2:  3= isn't valid elisp, so a
dedicated extension avoids read errors).
One file = one ERT test.

* Parser
State machine that uses =read= for s-expressions and switches to raw-text
capture after =(nalc-test-declare-stack)= / =(nalc-test-declare-output)=, collecting
lines until the next =(=.
Stack lines (=N:  expr=) are parsed with =math-read-expr=.

* Key semantics
- ="H"= in point specs = home (=.= line)
- Stack entries pushed bottom-first via =calc-push=
- Output compared with =math-equal= / =math-normalize=
- =nalc-test-declare-start-cond= / =nalc-test-declare-end-cond= forms are evaled and
  asserted

* ERT integration
=nalc-test-register-file= parses a =.nalc-test= file and evals an =ert-deftest= form.
=nalc-test-load-directory= is called from =test-init.el= so nalc-test tests run
alongside existing ERT tests with no change to the test runner invocation.

* Single new file
=my/calc/tests/nalc-test.el= — parser + runner + ERT registration.

* Example format

"Tests the calc-times command."

(nalc-test-declare-start-cond
  (null calc-hyperbolic-flag)
  (null calc-option-flag))

(nalc-test-declare-stack)
2:  3
1:  2
    .

(nalc-test-declare-start-point 'h)
(nalc-test-declare-command 'calc-times)
(nalc-test-declare-end-point 'h)

(nalc-test-declare-output)
1:  6
    .

(nalc-test-declare-end-cond
;; These optional conditions are checked at the end to verify the test.
)

** More
   - Automatically generates tests for keep-args.
   - Automatically generates tests for contextual modes (equation, line, subformula, etc).

** Notes
   `start-point` and `end-point` can take the following forms:
   - h (default value, the home area, same place point is moved after calc-align-stack-window)
   - h* (anywhere in the "home" area)
   # These values are contextualized to the affected line
   - ^ (start of entry)
   - ^ + 1 (start of entry plus 1 character )
   - $ (end of entry, past the last character)
   - $ - 2(end of entry minus 2 characters)
   - p (prefix area)
   - p + 1 (prefix area plus 1 character)
   - 4 (the fourth character, alias for ^ + 3)
   - 4 (the fourth character, alias for ^ + 3)
   # These values specify the line
   - 1:4 (the fourth character of the first line)
