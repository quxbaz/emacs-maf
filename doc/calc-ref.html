1. calc

1.1. (calc-load-everything)

(calc-load-everything)

Load all Calc extension modules. Ensures all Calc functions are available before calling them programmatically. 

(calc-load-everything)

2. Stack

2.1. Stack entry format

Each element of calc-stack is a list (formula lines selection):

  • car / index 0 — the full formula (a Lisp s-expression)
  • nth 1 — display line count (integer); initialized to 1 on push, updated to the actual rendered height by math-format-stack-value via calc-count-lines. Used by calc-substack-height for cursor positioning.
  • nth 2 — active selection sub-formula, or nil if none

Plain integers are replaced with (cplx n 0) by calc-prepare-selection so that duplicate small integers are distinct objects (eq-wise). They are restored to plain numbers by calc-normalize on next use.

Stack level 1 is the top.

2.2. (calc-get-stack-element)

(calc-get-stack-element X &optional SEL-MODE)

Low-level accessor. Given a raw stack entry X, return a part of it according to SEL-MODE:

SEL-MODE returns
nil selection (nth 2) if active and calc-use-selections; else formula (car)
'entry X itself
'sel (nth 2 x) — the selection sub-expression
'full (car x) — full formula, ignoring selection
other calc-sel-error

Called internally by calc-top and calc-top-list; rarely needed directly.

2.3. (calc-top)

(calc-top &optional N SEL-MODE)

Return the Nth stack entry (default 1 = top) according to SEL-MODE. Does not normalize. Use calc-top-n if normalization is wanted. 

(calc-top 1)          ;; formula at top (selection-aware)
(calc-top 1 'full)    ;; full formula, ignoring any active selection
(calc-top 1 'sel)     ;; active selection sub-expression
(calc-top 1 'entry)   ;; raw entry: (formula priority selection)

2.4. (calc-top-n)

(calc-top-n &optional N SEL-MODE)

Like calc-top, but normalizes the result and checks it is complete. Use when you need a canonical formula and don't mind the precision round-trip that calc-normalize may introduce. 

(calc-top-n 1)  ;; normalized formula at top

2.5. (calc-top-list)

(calc-top-list &optional N M SEL-MODE)

Return a list of N entries starting at stack level M (default both 1), in bottom-to-top order (level M+N-1 first, level M last). Does not normalize. SEL-MODE is the same as for calc-top. 

(calc-top-list 2 1)  ;; top two entries, bottom-to-top order

2.6. (calc-top-list-n)

(calc-top-list-n &optional N M SEL-MODE)

Like calc-top-list, but normalizes every entry. 

(calc-top-list-n 2 1)  ;; normalized top two entries

2.7. (calc-pop-push-record-list)

(calc-pop-push-record-list N PREFIX VALS &optional M SELS)

Replace N stack entries starting at level M (default 1) with VALS, and record the operation in the trail under PREFIX. The standard way to commit the result of a command: pop the inputs, push the result, leave a trail entry.

  • N — number of entries to pop.
  • PREFIX — trail label (string). If nil, no trail entry is recorded.
  • VALS — a list of formulas to push, or a single formula (auto-wrapped in a list). Pushed in order so the last element ends up at the top of the stack.
  • M — bottom-most stack level affected (default 1 = top).
  • SELS — optional list of selections paired with VALS.

If calc-any-selections is set and SELS is nil, falls through to calc-replace-selections instead of popping. Signals if the stack does not contain N + M - 1 entries. 

;; Pop 2, push their product, record under "mult"
(calc-pop-push-record-list 2 "mult" (list (calcFunc-mul (calc-top 2 'full)
                                                        (calc-top 1 'full))))

;; Single-value form: VALS may be the formula itself
(calc-pop-push-record-list 1 "double" (calcFunc-mul (calc-top 1 'full) 2))

3. Formula

3.1. (math-common-constant-factor) manual

(math-common-constant-factor EXPR)

Returns the constant factor in EXPR. 

(math-common-constant-factor '(+ (* 6 x) (* 4 y)))  ;; => 2
(math-common-constant-factor '(+ (* 3 x) y))        ;; => 1

3.2. (calc-encase-atoms) manual

(calc-encase-atoms X)

Modifies X in-place (via setcar) so every sub-formula is a unique atom, using the (cplx n 0) trick from calc-prepare-selection. 

(let ((x (list '+ 1 1)))
  (calc-encase-atoms x)
  x)  ;; => (+ (cplx 1 0) (cplx 1 0))  — two distinct objects

3.3. (math-expr-contains) manual

(math-expr-contains EXPR VAR)

Return non-nil if VAR occurs as a subexpression of EXPR. Uses equal (structural equality), unlike calc-find-sub-formula which uses eq. In f(a, a), both a=s are =equal but not eq, so this function would find either. 

(math-expr-contains '(+ x (* 2 y)) 'x)  ;; => t
(math-expr-contains '(+ x (* 2 y)) 'z)  ;; => nil

3.4. (math-expr-contains-count) manual

(math-expr-contains-count EXPR VAR)

Return the number of occurrences of VAR in EXPR, or nil if none. 

(math-expr-contains-count '(+ x (* 2 x)) 'x)  ;; => 2
(math-expr-contains-count '(+ x 1) 'z)         ;; => nil

3.5. (math-expr-depends) manual

(math-expr-depends EXPR VAR)

Return true if EXPR and VAR share any variable. 

(math-expr-depends '(+ x 1) 'y)  ;; => nil
(math-expr-depends '(+ x y) 'y)  ;; => t

3.6. (math-expr-contains-vars) manual

(math-expr-contains-vars EXPR)

Return true if EXPR contains any variables; nil if it contains only constants and functions with constant arguments. 

(math-expr-contains-vars '(+ x 1))  ;; => t
(math-expr-contains-vars '(+ 1 2))  ;; => nil

3.7. (math-expr-subst) manual

(math-expr-subst EXPR OLD NEW)

Return a copy of EXPR with all occurrences of OLD replaced by NEW. Handles lambda dummy variables correctly. 

(math-expr-subst '(+ x (* 2 x)) 'x 5)  ;; => (+ 5 (* 2 5))

3.8. (math-multi-subst) manual

(math-multi-subst EXPR OLD NEW)

Like math-expr-subst, but OLD and NEW are lists substituted simultaneously. If lists differ in length, trailing elements of the longer list are ignored. 

;; Swap x and y: sequential subst would give (+ y y), simultaneous gives (+ y x)
(math-multi-subst '(+ x y) '(x y) '(y x))  ;; => (+ y x)

4. Polynomials

4.1. (math-polynomial-p) manual

(math-polynomial-p EXPR VAR)

Return the degree of EXPR as a polynomial in VAR, or nil if it is not a polynomial. Degree is the highest power of VAR in the expanded form. A constant expression (no VAR) has degree 0. 

(math-polynomial-p '(+ (^ x 2) (* 3 x) 1) 'x)  ;; => 2
(math-polynomial-p '(calcFunc-sin x) 'x)         ;; => nil

4.2. (math-is-polynomial) manual

(math-is-polynomial EXPR VAR DEGREE LOOSE)

Return a coefficient list (a0 a1 ... an) for a0 + a1*var + ... + an*var^n, or nil if EXPR is not a polynomial in VAR. DEGREE caps the maximum accepted degree (recommended to avoid huge lists on inputs like (x+1)^1000).

If LOOSE is non-nil, coefficients may themselves depend on VAR as long as they are not polynomials; never returns nil in loose mode. 

(math-is-polynomial '(+ (* 2 x) 3) 'x nil nil)     ;; => (3 2)      i.e. 3 + 2x
(math-is-polynomial '(+ (^ x 2) -1) 'x nil nil)    ;; => (-1 0 1)   i.e. -1 + x^2

4.3. (math-polynomial-base) manual

(math-polynomial-base EXPR PRED)

Return a variable in EXPR such that EXPR is a polynomial in it, or nil. For multivariate expressions, the choice is arbitrary. PRED, if given, is called as (pred subexpr) with mpb-top-expr bound to the original EXPR; use it to add extra conditions or collect all suitable subexpressions. 

(math-polynomial-base '(+ (^ x 2) (* 3 x)) nil)  ;; => x

5. Simplification & rewriting

5.1. (math-simplify-extended) manual

(math-simplify-extended A)

Simplify A with additional rules that are more thorough but not always safe (e.g., simplifies sqrt(x^2) to x, valid only when x is positive). Implemented by binding math-living-dangerously to t and calling math-simplify. Dangerous rules check this variable before firing. 

(math-simplify-extended '(calcFunc-sqrt (^ x 2)))  ;; => x

5.2. (math-defsimplify) manual

(math-defsimplify FUNCS BODY)

Macro that registers a simplification rule, typically as a top-level form. FUNCS is a symbol (e.g., +, calcFunc-sqrt) or list of symbols; the rule applies to calls to any of those functions. BODY is written as a function body with expr bound to the matched call.

Return nil or the original expr to skip; any other value replaces expr. Rules are tried in definition order; the first match wins and restarts the loop. BODY must be plain Lisp — defmath conveniences are not available. The arguments of the matched call are already simplified before the rule runs, and may be rearranged destructively. 

;; arcsinh(-x) => -arcsinh(x)
(math-defsimplify calcFunc-arcsinh
  (and (math-looks-negp (nth 1 expr))
       (math-neg (list 'calcFunc-arcsinh (math-neg (nth 1 expr))))))

5.3. (math-apply-rewrites) manual

(math-apply-rewrites EXPR CRULES HEADS)

Apply compiled rewrite rule set CRULES to EXPR at the top level only, making at most one rewrite. Returns the rewritten expression, or nil if no rule matched or produced a change. HEADS is optional: a list of function names in EXPR used to narrow the search (the compiler tags each rule with its rarest function name). 

;; CRULES must be built with math-compile-rewrites first
(let ((rules (math-compile-rewrites '(vec (calcFunc-eq (^ a 2) (* a a))))))
  (math-apply-rewrites '(^ x 2) rules nil))  ;; => (* x x)

5.4. (math-match-patterns) manual

(math-match-patterns PAT VEC NOT-FLAG)

Return a new vector of elements from VEC that match (or, if NOT-FLAG is non-nil, don't match) any pattern in PAT. PAT is an uncompiled pattern set or pattern set variable. 

;; Keep only elements matching x^n
(math-match-patterns '(vec (^ a n)) '(vec (^ x 2) x (^ y 3)) nil)
;; => (vec (^ x 2) (^ y 3))

6. Solving

6.1. (math-solve-for) manual

(math-solve-for LHS RHS VAR FULL)

Solve LHS = RHS for VAR, returning the right-hand side, or nil if unsolvable. VAR must appear in LHS or RHS. The return value does not contain VAR (unlike the user-level solve=/=finv). If FULL is non-nil, includes dummy signs and integers. 

;; 2x + 3 = 7  =>  x = 2
(math-solve-for '(+ (* 2 x) 3) 7 'x nil)  ;; => 2

Inverses for custom functions are registered via a math-inverse property. Use (math-solve-get-sign x) and (math-solve-get-int x) to introduce arbitrary sign/integer variables; these return x unchanged in non-full mode.

6.2. (math-solve-eqn) manual

(math-solve-eqn EXPR VAR FULL)

Like math-solve-for, but EXPR is an equation or inequality. A bare expression is treated as EXPR = 0. Returns an equation or inequality, or nil. 

(math-solve-eqn '(calcFunc-eq (+ x 3) 7) 'x nil)  ;; => (calcFunc-eq x 4)
(math-solve-eqn '(+ x 3) 'x nil)                   ;; => (calcFunc-eq x -3)

6.3. (math-solve-system) manual

(math-solve-system EXPRS VARS FULL)

Solve a system of equations. EXPRS and VARS are typically equal-length vectors. 

;; x + y = 3, x - y = 1  =>  x = 2, y = 1
(math-solve-system '(vec (calcFunc-eq (+ x y) 3)
                         (calcFunc-eq (- x y) 1))
                   '(vec x y) nil)
;; => (vec (calcFunc-eq x 2) (calcFunc-eq y 1))

7. Selection

7.1. (calc-prepare-selection) manual

(calc-prepare-selection &optional NUM)

Prepare a stack entry for selection operations. NUM specifies which entry to use; if omitted, the entry containing the cursor is used.

Stores results in: calc-selection-cache-num (the entry number used), calc-selection-cache-entry (the entry as a list, as calc-top-list would return with entry as the selection mode), and calc-selection-cache-comp (a "tagged" composition mapping cursor positions to sub-formulas; see I/O and Formatting Functions).

A complication: formulas can contain duplicate small integers. In [1, 2, 1], both 1=s are =eq in Lisp, so storing 1 as a selection would falsely match both. To prevent this, calc-prepare-selection replaces plain integers with (cplx n 0) lists — displayed identically but distinct objects, so no two sub-formulas are eq. The next time the entry is used in a computation, calc-normalize restores them to plain numbers. 

(calc-prepare-selection)    ;; prepare entry under cursor
(calc-prepare-selection 2)  ;; prepare stack entry 2

7.2. (calc-find-selected-part) manual

(calc-find-selected-part)

Find the smallest sub-formula containing the cursor. Assumes calc-prepare-selection has been called first. Returns nil if the cursor is not on any part of the formula. 

(calc-prepare-selection)
(calc-find-selected-part)  ;; => sub-formula under cursor, or nil

7.3. (calc-find-sub-formula) manual

(calc-find-sub-formula EXPR PART)

Search EXPR for PART by object identity (eq). Returns:

  • t — EXPR itself is PART
  • N — PART is found within the Nth argument of EXPR (1-based)
  • nil — PART is not in EXPR

The index N can be passed to calc-find-nth-part to retrieve the child. Does not account for associativity. 

(let* ((expr '(+ a (* b c)))
       (sub  (nth 2 expr)))            ;; (* b c)
  (calc-find-sub-formula expr sub))    ;; => 2

7.4. (calc-locate-cursor-element)

(calc-locate-cursor-element PT)

Return the stack index of the entry whose display region contains buffer position PT. The result is suitable for passing to calc-cursor-stack-index. 

(calc-locate-cursor-element (point))  ;; => stack index at cursor position

8. Display

8.1. (math-format-value) manual

(math-format-value A &optional W)

Format A as a display string. For plain scalars in nil/flat/unform language modes, delegates to math-format-number. Otherwise composes A and converts to string via math-composition-to-string with no line breaking. W optionally limits the output width. 

(math-format-value '(+ x 1))   ;; => "x + 1"
(math-format-value '(^ x 2))   ;; => "x^2"

8.2. (math-format-stack-value)

(math-format-stack-value ENTRY)

Format a complete stack entry for display in the Calc buffer. ENTRY is a list (value lines selection). Handles selection highlighting, left label (calc-left-label), and line numbering (calc-line-numbering). Sets calc-selection-cache-entry and calc-any-selections as side effects. 

;; ENTRY format: (value display-lines selection)
(math-format-stack-value (list '(+ x 1) nil nil))

8.3. (calc-align-stack-window)

(calc-align-stack-window)

Scroll the Calc stack window so the home line (.) is visible near the bottom. Positions the window start accordingly and moves point to the home line. If called from outside the Calc buffer, switches to it first. 

(calc-align-stack-window)
