#+title: Calc Reference

* Formula

  =(math-common-constant-factor EXPR)=
  Returns the *constant* factor in `EXPR`.

  =(calc-encase-atoms X)=
  Modifies `X` in-place (via `setcar`) so every sub-formula is a unique atom,
  using the `(cplx n 0)` trick from `calc-prepare-selection`.

  =(math-expr-contains EXPR VAR)=
  Return non-nil if `VAR` occurs as a subexpression of `EXPR`. Uses `equal`
  (structural equality), unlike `calc-find-sub-formula` which uses `eq`. In
  `f(a, a)`, both `a`s are `equal` but not `eq`, so this function would
  find either.

  =(math-expr-contains-count EXPR VAR)=
  Return the number of occurrences of `VAR` in `EXPR`, or nil if none.

  =(math-expr-depends EXPR VAR)=
  Return true if `EXPR` and `VAR` share any variable.

  =(math-expr-contains-vars EXPR)=
  Return true if `EXPR` contains any variables; nil if it contains only
  constants and functions with constant arguments.

  =(math-expr-subst EXPR OLD NEW)=
  Return a copy of `EXPR` with all occurrences of `OLD` replaced by `NEW`.
  Handles lambda dummy variables correctly.

  #+begin_src emacs-lisp :results silent
  (math-expr-subst '(+ x (* 2 x)) 'x 5)  ;; => (+ 5 (* 2 5))
  #+end_src

  =(math-multi-subst EXPR OLD NEW)=
  Like `math-expr-subst`, but `OLD` and `NEW` are lists substituted
  simultaneously. If lists differ in length, trailing elements of the longer
  list are ignored.

  #+begin_src emacs-lisp :results silent
  ;; Swap x and y: sequential subst would give (+ y y), simultaneous gives (+ y x)
  (math-multi-subst '(+ x y) '(x y) '(y x))  ;; => (+ y x)
  #+end_src

* Polynomials

  =(math-polynomial-p EXPR VAR)=
  Return the degree of `EXPR` as a polynomial in `VAR`, or nil if it is not
  a polynomial. Degree is the highest power of `VAR` in the expanded form. A
  constant expression (no `VAR`) has degree 0.

  =(math-is-polynomial EXPR VAR DEGREE LOOSE)=
  Return a coefficient list `(a0 a1 ... an)` for `a0 + a1*var + ... + an*var^n`,
  or nil if `EXPR` is not a polynomial in `VAR`. `DEGREE` caps the maximum
  accepted degree (recommended to avoid huge lists on inputs like `(x+1)^1000`).

  If `LOOSE` is non-nil, coefficients may themselves depend on `VAR` as long
  as they are not polynomials; never returns nil in loose mode.

  #+begin_src emacs-lisp :results silent
  (math-is-polynomial '(+ (* 2 x) 3) 'x nil nil)     ;; => (3 2)      i.e. 3 + 2x
  (math-is-polynomial '(+ (^ x 2) -1) 'x nil nil)    ;; => (-1 0 1)   i.e. -1 + x^2
  #+end_src

  =(math-polynomial-base EXPR PRED)=
  Return a variable in `EXPR` such that `EXPR` is a polynomial in it, or nil.
  For multivariate expressions, the choice is arbitrary. `PRED`, if given, is
  called as `(pred subexpr)` with `mpb-top-expr` bound to the original `EXPR`;
  use it to add extra conditions or collect all suitable subexpressions.

* Simplification & rewriting

  =(math-simplify-extended A)=
  Simplify `A` with additional rules that are more thorough but not always safe
  (e.g., simplifies `sqrt(x^2)` to `x`, valid only when `x` is positive).
  Implemented by binding `math-living-dangerously` to `t` and calling
  `math-simplify`. Dangerous rules check this variable before firing.

  =(math-defsimplify FUNCS BODY)=
  Macro that registers a simplification rule, typically as a top-level form.
  `FUNCS` is a symbol (e.g., `+`, `calcFunc-sqrt`) or list of symbols; the rule
  applies to calls to any of those functions. `BODY` is written as a function
  body with `expr` bound to the matched call.

  Return nil or the original `expr` to skip; any other value replaces `expr`.
  Rules are tried in definition order; the first match wins and restarts the
  loop. `BODY` must be plain Lisp ã `defmath` conveniences are not available.
  The arguments of the matched call are already simplified before the rule
  runs, and may be rearranged destructively.

  #+begin_src emacs-lisp :results silent
  ;; arcsinh(-x) => -arcsinh(x)
  (math-defsimplify calcFunc-arcsinh
    (and (math-looks-negp (nth 1 expr))
         (math-neg (list 'calcFunc-arcsinh (math-neg (nth 1 expr))))))
  #+end_src

  =(math-apply-rewrites EXPR CRULES HEADS)=
  Apply compiled rewrite rule set `CRULES` to `EXPR` at the top level only,
  making at most one rewrite. Returns the rewritten expression, or nil if no
  rule matched or produced a change. `HEADS` is optional: a list of function
  names in `EXPR` used to narrow the search (the compiler tags each rule with
  its rarest function name).

  =(math-match-patterns PAT VEC NOT-FLAG)=
  Return a new vector of elements from `VEC` that match (or, if `NOT-FLAG` is
  non-nil, don't match) any pattern in `PAT`. `PAT` is an uncompiled pattern
  set or pattern set variable.

* Solving

  =(math-solve-for LHS RHS VAR FULL)=
  Solve `LHS` = `RHS` for `VAR`, returning the right-hand side, or nil if
  unsolvable. `VAR` must appear in `LHS` or `RHS`. The return value does not
  contain `VAR` (unlike the user-level `solve`/`finv`). If `FULL` is non-nil,
  includes dummy signs and integers.

  #+begin_src emacs-lisp :results silent
  ;; 2x + 3 = 7  =>  x = 2
  (math-solve-for '(+ (* 2 x) 3) 7 'x nil)  ;; => 2
  #+end_src

  Inverses for custom functions are registered via a `math-inverse` property.
  Use `(math-solve-get-sign x)` and `(math-solve-get-int x)` to introduce
  arbitrary sign/integer variables; these return `x` unchanged in non-full mode.

  =(math-solve-eqn EXPR VAR FULL)=
  Like `math-solve-for`, but `EXPR` is an equation or inequality. A bare
  expression is treated as `EXPR` = 0. Returns an equation or inequality, or nil.

  =(math-solve-system EXPRS VARS FULL)=
  Solve a system of equations. `EXPRS` and `VARS` are typically equal-length
  vectors.

* Selection

  =(calc-prepare-selection &optional NUM)=
  Prepare a stack entry for selection operations. `NUM` specifies which entry
  to use; if omitted, the entry containing the cursor is used.

  Stores results in: `calc-selection-cache-num` (the entry number used),
  `calc-selection-cache-entry` (the entry as a list, as `calc-top-list` would
  return with entry as the selection mode), and `calc-selection-cache-comp`
  (a "tagged" composition mapping cursor positions to sub-formulas; see I/O
  and Formatting Functions).

  A complication: formulas can contain duplicate small integers. In `[1, 2, 1]`,
  both `1`s are `eq` in Lisp, so storing `1` as a selection would falsely match
  both. To prevent this, `calc-prepare-selection` replaces plain integers with
  `(cplx n 0)` lists ã displayed identically but distinct objects, so no two
  sub-formulas are `eq`. The next time the entry is used in a computation,
  `calc-normalize` restores them to plain numbers.

  =(calc-find-selected-part)=
  Find the smallest sub-formula containing the cursor. Assumes
  `calc-prepare-selection` has been called first. Returns nil if the cursor
  is not on any part of the formula.

  =(calc-find-sub-formula EXPR PART)=
  Search `EXPR` for `PART` by object identity (`eq`). Returns:
  - `t` ã `EXPR` itself is `PART`
  - N ã `PART` is found within the Nth argument of `EXPR` (1-based)
  - `nil` ã `PART` is not in `EXPR`

  The index N can be passed to `calc-find-nth-part` to retrieve the child.
  Does not account for associativity.

  #+begin_src emacs-lisp :results silent
  (let* ((expr '(+ a (* b c)))
         (sub  (nth 2 expr)))            ;; (* b c)
    (calc-find-sub-formula expr sub))    ;; => 2
  #+end_src

  =(calc-locate-cursor-element PT)=
  Return the stack index of the entry whose display region contains buffer
  position `PT`. The result is suitable for passing to `calc-cursor-stack-index`.

* Display

  =(math-format-value A &optional W)=
  Format `A` as a display string. For plain scalars in nil/flat/unform language
  modes, delegates to `math-format-number`. Otherwise composes `A` and converts
  to string via `math-composition-to-string` with no line breaking. `W`
  optionally limits the output width.

  =(math-format-stack-value ENTRY)=
  Format a complete stack entry for display in the Calc buffer. `ENTRY` is a
  list `(value lines selection)`. Handles selection highlighting, left label
  (`calc-left-label`), and line numbering (`calc-line-numbering`). Sets
  `calc-selection-cache-entry` and `calc-any-selections` as side effects.

  =(calc-align-stack-window)=
  Scroll the Calc stack window so the home line (.) is visible near the
  bottom. Positions the window start accordingly and moves point to the home
  line. If called from outside the Calc buffer, switches to it first.

* Loading

  =(calc-load-everything)=
  Load all Calc extension modules. Ensures all Calc functions are available
  before calling them programmatically.
